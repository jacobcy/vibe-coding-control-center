# publish-report (after implement)

When the implementation phase completes, record the run report (e.g.
`docs/reports/<issue>-report.md`) as `report_ref` for the current Vibe flow.

## Publish

Reports are not generated by spec-kit itself; pass the report path explicitly
via the adapter's `--artifact` flag:

```bash
.specify/extensions/vibe-spec-bridge/hooks/publish-artifact.sh report \
  --artifact docs/reports/<issue>-report.md --branch <branch>
```

Or call the public writer directly:

```bash
vibe3 handoff report docs/reports/<issue>-report.md --branch <branch>
```

## Contract

- MUST use `vibe3 handoff report` — never write shared state directly (FR-016).
- The report path MUST be a worktree-relative canonical doc; paths under
  `temp/logs/` or the shared handoff store are rejected.
- Re-publishing the same path is safe (FR-018).
