# publish-audit (after review)

When the review phase completes, record the audit document (e.g.
`docs/reports/<issue>-audit.md`) as `audit_ref` for the current Vibe flow.

## Publish

Audits are not generated by spec-kit itself; pass the audit path explicitly
via the adapter's `--artifact` flag:

```bash
.specify/extensions/vibe-spec-bridge/hooks/publish-artifact.sh audit \
  --artifact docs/reports/<issue>-audit.md --branch <branch>
```

Or call the public writer directly:

```bash
vibe3 handoff audit docs/reports/<issue>-audit.md --branch <branch>
```

## Contract

- MUST use `vibe3 handoff audit` — never write shared state directly (FR-016).
- The audit path MUST be a worktree-relative canonical doc; paths under
  `temp/logs/` or the shared handoff store are rejected.
- Re-publishing the same path is safe (FR-018).
