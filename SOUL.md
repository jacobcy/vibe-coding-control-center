# VIBE Constitution

Vibe Coding 的核心是：**认知优先，代码次之**。

## 1. 项目身份
- 目标：提升 AI 协作开发效率与工程可维护性。
- 定位：编排工具与流程治理，不重造 Agent 本体。

## 2. 两阶段方法
- 探索期：快速试错，避免过早重规范。
- 收敛期：启用强治理（范围、LOC、PR 合规、回归验证）。
- 阶段切换依据：是否已形成稳定心智模型（可通过审计确认）。

## 3. Agent 行为准则
- 思考语言：英文。
- 对外输出：中文。
- 原则：最小正确改动、上下文先行、可验证交付。

## 4. 不可协商边界
- 不直接在 `main/master` 上开发。
- 仅修改仓库内文件，不触碰系统级配置。
- 不做与当前任务无关的重构。

## 5. 质量底线
- 变更必须可运行、可验证。
- 失败要快速暴露（fail fast），不要静默吞错。
- 交付物应可审阅、可复现。

## 6. 权限层级
当规则冲突时，优先级：`SOUL.md` > `CLAUDE.md` > `.agent/rules/*` > 其他约定。

---
以“保护认知资产”为最高目标，治理代码膨胀与流程失控。
