# VIBE Constitution

Vibe Coding 的核心是：**认知优先，代码次之**。

## 0. 单一事实原则（Single Source of Truth）

**核心原则**：每个概念、规则、流程只在一个文档中详细阐述，其他文档只能引用，不能重复阐述。

与单一事实原则配套的原则是：**渐进披露（Progressive Disclosure）**。

**渐进披露原则**：
- 入口文件只保留导航、最小必要约束和引用链
- 标准文件只保留项目内部稳定规范，不承载临时讨论和执行细节
- 规则文件承载执行细则、模式和 agent 行为约束
- 参考文件只承载外部知识、外部库和外部资料引用
- 不允许在单一文件中堆叠复杂规范、完整流程、术语解释和实现细节
- 复杂内容必须按层次拆开，并通过引用串联

**控制目标**：
- 控制单一文件认知负担和 token 体积
- 让人类和 agent 都能沿“入口 -> 标准/规则 -> 细节”路径逐层进入
- 避免入口文件和标准文件重新膨胀成第二份总手册

**文档职责分工**：

| 文件 | 职责 | 受众 | 可以写什么 | 不能写什么 |
|------|------|------|-----------|-----------|
| **SOUL.md** | 项目宪法和核心原则 | 全员 | 价值观、边界、优先级 | 具体实现、技术细节 |
| **STRUCTURE.md** | 项目结构定义 | 全员 | 目录结构、文件职责 | 开发流程、使用方法 |
| **AGENTS.md** | AI Agent 入口 | AI | 导航、快速开始 | 详细规则（只引用） |
| **CLAUDE.md** | 项目上下文和硬规则 | AI | 技术栈、命令、硬规则 | 原则（引用 SOUL） |
| **README.md** | 项目介绍 | 用户 | 功能、快速开始 | 开发细节（引用 DEVELOPER） |
| **DEVELOPER.md** | 开发指南 | 开发者 | 开发流程、工具使用 | 原则（引用 SOUL） |
| **docs/README.md** | 文档结构 | 全员 | 文档组织、导航 | 具体文档内容 |
| **入口文件** | 进入项目的第一跳文档 | 人类/AI | 导航、最小必要约束、引用链 | 复杂规范、细节规则、重复定义 |
| **标准文件** | 项目内部稳定规范真源 | 人类/AI | 标准、边界、正式定义 | 过程噪音、临时讨论、实现细节 |
| **参考文件** | 外部知识与外部资料引用 | 人类/AI | 外部库、外部文档、外部最佳实践 | 项目内部规范真源 |
| **规则文件** | agent 执行规则与模式细则 | AI | 执行约束、实现细则、模式 | 项目总原则、外部参考百科 |

**引用规则**：
- ✅ 允许：`详见 [SOUL.md](SOUL.md)`、`参考 [CLAUDE.md](CLAUDE.md) §HARD RULES`
- ❌ 禁止：在多个文件中重复阐述同一规则或概念
- ❌ 禁止：在非权威文件中详细解释其他文件的内容

**维护原则**：
- 修改概念时，只修改权威文件
- 其他文件的引用自动保持一致
- 定期审查，消除重复内容
- 若某内容已能通过引用获得，不得在更高层文件重复展开
- 入口文件和标准文件发现膨胀时，应优先下沉到规则文件或更合适的真源文件

## 1. 项目身份
- 目标：提升 AI 协作开发效率与工程可维护性。
- 定位：编排工具与流程治理，不重造 Agent 本体。

## 2. 两阶段方法
- 探索期：快速试错，避免过早重规范。
- 收敛期：启用强治理（范围、LOC、PR 合规、回归验证）。
- 阶段切换依据：是否已形成稳定心智模型（可通过审计确认）。

## 3. Agent 行为准则
- 思考语言：英文。
- 对外输出：中文。
- 原则：最小正确改动、上下文先行、可验证交付。

## 4. 不可协商边界
- 不直接在 `main/master` 上开发。
- 仅修改仓库内文件，不触碰系统级配置。
- 不做与当前任务无关的重构。

## 5. 质量底线
- 变更必须可运行、可验证。
- 失败要快速暴露（fail fast），不要静默吞错。
- 交付物应可审阅、可复现。

## 6. 权限层级
当规则冲突时，优先级：`SOUL.md` > `CLAUDE.md` > `.agent/rules/*` > 其他约定。

---
以“保护认知资产”为最高目标，治理代码膨胀与流程失控。

## 7. 文档维护责任

| 文档 | 维护者 | 审查频率 | 变更触发条件 |
|------|--------|---------|-------------|
| `SOUL.md` | 项目负责人 | 每年 | 核心原则变更 |
| `STRUCTURE.md` | 项目负责人 | 每次结构变更 | 目录/文件结构变化 |
| `AGENTS.md` | 项目负责人 | 每季度 | 必读文档列表变化 |
| `CLAUDE.md` | 项目负责人 | 每次规则变更 | 硬规则、技术栈变化 |
| `README.md` | 项目负责人 | 每次功能变更 | 用户可见功能变化 |
| `DEVELOPER.md` | 开发负责人 | 每次流程变更 | 开发流程、工具变化 |
| `docs/README.md` | 文档负责人 | 每次文档结构变更 | 文档目录结构变化 |
