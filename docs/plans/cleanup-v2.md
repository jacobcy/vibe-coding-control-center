# 清理 V2 Shell 代码计划

## 目标

清理项目中遗留的 V2 Shell 代码，保留 V3 Python 架构。

## 清理范围

### 1. Shell 脚本（bin/）
- **保留**: `vibe`（主入口脚本）、`vibe3`（V3 入口）
- **检查**: 其他脚本是否仍在使用

### 2. Shell 库（lib/）
- **检查**: 每个库文件的使用情况
- **移除**: 未使用的 V2 专用库
- **保留**: 仍在使用的库

### 3. 测试（tests/vibe2/）
- **移除**: 所有 Bats 测试
- **保留**: V3 pytest 测试

### 4. 配置文件
- **清理**: V2 相关配置
- **更新**: .gitignore

### 5. 文档
- **更新**: 文档引用
- **移除**: V2 相关说明

## 实施步骤

### Phase 1: 分析依赖关系
1. 检查每个 lib/ 文件的使用情况
2. 检查每个 bin/ 脚本的使用情况
3. 确定哪些可以安全移除

### Phase 2: 清理未使用文件
1. 移除未使用的 lib/ 文件
2. 移除未使用的 bin/ 脚本
3. 移除 tests/vibe2/ 目录

### Phase 3: 更新入口脚本
1. 更新 bin/vibe 脚本，移除 V2 调用
2. 确保 V3 入口正常工作

### Phase 4: 更新文档和配置
1. 更新相关文档
2. 清理配置文件
3. 更新 .gitignore

### Phase 5: 测试验证
1. 运行 V3 测试
2. 验证功能正常

## 注意事项

- 保留必要的入口脚本
- 确保不破坏现有功能
- 更新相关文档
- 提交前充分测试
