## Findings

### Current Status

1. `skills/vibe-save/SKILL.md` 已移除把只读 shell 查询描述成“自动对齐”的表述，并改为从共享真源读取当前 task 事实。
2. `skills/vibe-continue/SKILL.md`、`skills/vibe-save/SKILL.md` 与 `.agent/workflows/vibe-start.md` 现已显式对齐 standards 引用，不再各自定义第二套 task/worktree/plan 语义。
3. `skills/vibe-continue/SKILL.md` 已收敛为“共享真源优先，`task.md` 仅作 handoff 补充”，不再把 `.vibe/current-task.json`、`.vibe/focus.md`、`.vibe/session.json` 作为恢复入口。
4. `.agent/workflows/vibe-start.md` 已限定执行图纸来源为显式计划文件或 task 元数据中的 `plan_path`，不再把 `.agent/context/task.md` 当作正式执行 checklist。
5. shell 实现已同步收敛：`lib/task_write.sh` 与 `lib/task_actions.sh` 不再生成 worktree 本地 `.vibe/*`，`lib/task_query.sh` 不再读取 `.vibe/current-task.json`。

### Remaining Gaps In Scope

1. 本审计范围内未发现继续依赖 worktree 本地 `.vibe/*` 的现行 shell 入口；历史 references 仅应保留在 archive 或审计证据语境中。

## 修复原则

> 修复目标是让 skill 或 workflow 回归真源，不是让真源去适配 skill 文本。

1. 删除所有把只读 shell 查询包装成“自动对齐”“自动修复”的表述。
2. 为 continue/save/start 补齐与其语义范围匹配的 standards 引用，不在 skill/workflow 内重复定义术语。
3. 对尚无执行器的 hook 与尚无入口的 `wtinit`，统一改写为 capability gap 或待人工触发说明，不再写成既有能力。
4. 明确 `.agent/context/task.md` 只是本地 handoff/草稿缓存；正式执行图纸应来自 plan 类文档或显式任务计划，而不是临时缓存。
