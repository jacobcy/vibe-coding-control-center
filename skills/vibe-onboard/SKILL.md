# /vibe-onboard - 安装后入门引导

Vibe Center 安装完成后的一站式引导工具，帮助用户完成全部配置、检查系统状态、了解核心功能，确保系统就绪可用。

**完成后状态**：所有基础配置已完成，系统状态健康，用户了解核心用法，可以开始使用 Vibe Center 开发。

---

## 核心职责

1. **环境诊断检查**：运行 `vibe doctor` 全面检查系统状态，自动修复可解决的问题
2. **密钥配置引导**：引导用户配置必要的 API 密钥（OpenAI/Anthropic/GitHub 等）
3. **常用功能介绍**：介绍 Vibe Center 的核心功能和常用命令
4. **可选组件安装**：引导用户安装需要的可选组件（direnv、pre-commit、supervisor 服务等）
5. **快速开始教程**：给出简单的使用示例，帮助用户快速上手

---

## 停止点

完成后输出：
- ✅ 系统状态检查完成，所有依赖正常
- ✅ 密钥配置已完成
- ✅ 可选组件已按需安装
- **下一步**：开始使用 Vibe Center，运行 `/vibe-new` 创建新任务

---

## 完整流程

```
/vibe-onboard
  ├─ Step 1: 环境诊断与检查
  │   ├─ 运行 `vibe doctor` 全面扫描系统状态
  │   ├─ 自动修复可解决的问题（依赖缺失、权限问题等）
  │   └─ 对于无法自动修复的问题，给出明确的解决方案
  │
  ├─ Step 2: 密钥配置引导
  │   ├─ 检查 `~/.vibe/keys.env` 中的密钥配置
  │   ├─ 列出缺失的必要密钥，引导用户输入配置
  │   └─ 验证密钥有效性
  │
  ├─ Step 3: 核心功能介绍
  │   ├─ 介绍 Vibe Center 的核心定位：AI 开发编排工具
  │   ├─ 展示常用命令：
  │   │   - `/vibe-new`：创建新开发任务
  │   │   - `/vibe-commit`：提交代码并生成规范 commit 信息
  │   │   - `/vibe-review`：代码评审
  │   │   - `vibe doctor`：系统诊断
  │   │   - `vibe keys`：密钥管理
  │   └─ 介绍项目架构和扩展能力
  │
  ├─ Step 4: 可选组件安装引导
  │   ├─ 询问用户是否需要安装以下可选组件：
  │   │   - direnv：自动加载项目环境变量
  │   │   - pre-commit hooks：代码提交前自动检查
  │   │   - supervisor 后台服务：启动 orchestrator 等后台服务
  │   └─ 为用户按需安装配置
  │
  └─ Step 5: 快速开始教程
      ├─ 给出简单的使用示例：如何创建一个新任务，如何提交代码
      ├─ 说明文档位置：`CLAUDE.md`、`docs/` 目录
      └─ 提示用户有问题可以随时使用 `/vibe-help` 寻求帮助
```

---

## 注意事项

1. 不要强制用户安装不需要的可选组件，每一步都需要用户确认
2. 密钥输入时要保证安全，不要记录或泄露用户的密钥信息
3. 对于有问题的检查项，要给出明确、可执行的解决方案，不要含糊其辞
4. 引导过程要友好，对于不熟悉技术的用户也要容易理解
