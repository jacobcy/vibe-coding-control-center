# Vibe Coding Control Center - Next Generation

> 一个现代化、高度可扩展的 AI 开发工具管理套件，专注于开发者生产力和安全性。

## 🚀 核心特性

- **模块化架构** - 高度解耦的模块系统，易于扩展和维护
- **企业级安全** - 全面的输入验证、路径清理和安全文件操作
- **国际化支持** - 内建多语言支持
- **智能缓存系统** - 带有过期机制的高性能缓存
- **插件系统** - 可扩展的插件架构
- **健壮错误处理** - 带有重试和断路器的高级错误处理

## 🛠️ 技术亮点

### 1. 模块化设计
```bash
lib/
├── utils.sh           # 核心工具函数（已增强安全性）
├── config.sh          # 集中化配置管理
├── plugins.sh         # 插件系统
├── cache.sh           # 高性能缓存系统
├── error_handling.sh  # 带重试和断路器的错误处理
├── i18n.sh           # 国际化支持
└── testing.sh        # 测试框架
```

### 2. 安全强化
- 修复了 `prompt_user` 函数中的 `eval` 命令注入漏洞
- 增强的输入验证和路径清理
- 安全的文件操作和配置管理

### 3. 版本管理改进
- 更准确的版本比较算法
- 版本标准化和归一化功能
- 版本列表最大值计算

## 📦 快速开始

### 现代化安装
```bash
# 使用新的安装脚本
./scripts/install.sh

# 或者直接使用主控制台
./scripts/vibecoding.sh
```

### 项目初始化
```bash
# 初始化新项目
./install/init-project.sh [项目名称]
```

## 🔧 开发指南

### 运行完整测试
```bash
./scripts/test-all.sh
```

### 使用新功能
- **配置管理**：使用 `config_get` 和 `config_set` 进行配置操作
- **缓存系统**：使用 `cache_set` 和 `cache_get` 进行缓存操作
- **国际化**：使用 `translate` 函数进行字符串本地化
- **插件系统**：使用 `register_plugin` 和 `load_plugin` 管理插件

## 🌐 国际化支持

系统内置支持多语言：
- English (en)
- 中文 (zh)
- 日本語 (ja)
- 한국어 (ko)

通过 `set_locale` 函数切换语言环境。

## 🔒 安全特性

- 输入验证：所有用户输入都会经过严格验证
- 路径验证：防止路径遍历攻击
- 安全文件操作：防止文件系统攻击
- 错误隔离：防止错误信息泄露敏感数据

## 🏗️ 架构改进

### 1. 依赖注入系统

### 2. 插件架构
通过 `plugins.sh` 实现可扩展的插件系统。

### 3. 缓存层
通过 `cache.sh` 实现带过期策略的缓存机制。

### 4. 健壮错误处理
通过 `error_handling.sh` 实现重试和断路器模式。
